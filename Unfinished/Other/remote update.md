# Передача изменений на удаленный компьютер

## Самый простой вариант — общая папка через RDP

Подходит, если нужно «прямо сейчас» и без сложной настройки.

Как работает

RDP умеет пробрасывать локальные диски на удалённый ПК.

Что сделать

При подключении в mstsc.exe:

Показать параметры → Локальные ресурсы → Подробнее

Поставь галочку на нужный диск (например, C: или отдельную папку)

На удалённом ПК диск будет виден как:

\\\tsclient\C


Просто копируешь обновлённые плагины

Плюсы

- Очень быстро
- Никаких сторонних инструментов
- Работает «из коробки»

Минусы

- Копирование вручную
- Нет автоматизации
- Windows Home — не поддерживает приём RDP-подключений (только клиент).
- Не просто подключиться к другому компу. набираешь ip и ничего не происходит.

## Скрипт с копированием файлов

```powershell
robocopy "C:\Plugins" "\\REMOTE-PC\C$\Plugins" /MIR /XD bin obj .git
```

## Профессиональный вариант — Git

код размещать в одном репозитории, а готовый проект в другом.

### Вариант 1. Git по обычной сетевой папке (самый простой)

Лучше всего для Windows + RDP.

Схема

```perl
Твой ПК → push → bare-репозиторий в сетевой папке → pull → удалённый ПК
```

Как настроить
1. На одном из компьютеров (например, удалённом)

Создай bare-репозиторий:

git init --bare D:\git\plugins.git

2. Расшарь папку D:\git

Например, как:

\\REMOTE-PC\git

3. На локальном ПК

Добавь remote:

git remote add lan \\REMOTE-PC\git\plugins.git


Пуш:

git push lan main

4. На удалённом ПК (где плагины реально используются)
git pull lan main

Плюсы

✔ Очень просто
✔ Не нужен сервер
✔ Работает чисто по LAN
✔ Отлично для RDP

Минусы

✖ Windows-шары не самые быстрые
✖ Нет прав/аутентификации Git (используются права Windows)

🔒 Вариант 2. Git по SSH (самый правильный)

Рекомендую, если будешь этим пользоваться постоянно.

Что нужно

Включён SSH-сервер на удалённом ПК

Windows: Optional Features → OpenSSH Server

SSH-ключи

Схема
Твой ПК → git push ssh → удалённый ПК → git pull

Настройка (коротко)
ssh-keygen
ssh user@remote-pc


Создай репозиторий:

git init --bare /repos/plugins.git


Remote:

git remote add lan user@remote-pc:/repos/plugins.git


Пуш:

git push lan main

Плюсы

✔ Быстро
✔ Безопасно
✔ Классический Git-подход
✔ Работает в чистом LAN

Минусы

✖ Чуть больше настройки

🧠 Вариант 3. Pull автоматически при каждом пуше

Если хочешь максимально оперативно.

Идея

На удалённом ПК:

git pull по таймеру

или post-receive hook

После пуша плагин сам обновляется

Пример hook (bare repo)
hooks/post-receive

#!/bin/sh
cd /app/plugins
git pull lan main


⚠️ На Windows это делается через .cmd или PowerShell.

🧩 Что лучше выбрать
Сценарий	Рекомендация
Хочу быстро и просто	Сетевая папка
Хочу правильно и надолго	SSH
Хочу автообновление	SSH + hook
💡 Рекомендация именно для плагинов

Так как ты:

пишешь плагины

работаешь через RDP

хочешь быстро передавать изменения

👉 Git по LAN + post-build / post-receive — идеальный вариант.
